kepler-skills

Stream task operations toolkit for Kepler streaming engine

Capabilities

▶ Query Task

Inspect task SQL, configuration parameters, parallelism settings, partition info, and recent operations.

⟳ Tune Task

Auto-align worker size and parallelism with upstream partitions. Configure memory, buffer, CPU slots.

⏻ Start / Stop / Restart

Control task lifecycle with graceful restart (preserve state) or immediate stop.

↺ Reset Task

Clear task state and restart from specified offset timestamp.

☰ Operation Logs

View recent operations: who did what and when.

⬡ Multi-Site Support

Switch between Kepler deployments. Auth credentials isolated per-site.

API Endpoints

Endpoint Method Description
/api/sql/{topologyId} GET Query task SQL and config
/api/topology/topology/offset/{id} GET Query partition count and delay
/api/oprLogs/list GET Query operation history
/api/job/simpleQuery POST Query task status and offset
/api/sql/update POST Update task configuration
/api/topology/{id}/ops/start POST Start stopped task
/api/topology/{id}/ops/restart POST Restart running task
/api/topology/{id}/ops/stop POST Stop task
/api/job/batchOpr POST Reset task to specific offset
Authentication: AccessKey signature (MD5) or Cookie. Credentials stored per-site with 600 permissions.

Task Status

ACTIVE Running and processing data STOP Stopped, can be started or reset RESTARTING In transition
