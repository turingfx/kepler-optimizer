🧩 组件详解

🔍dg.sh (诊断主控)

智能诊断入口脚本，自动决策执行哪种优化策略

  • 自动采集任务指标
  • 检查并发一致性
  • 决策优化路径
  • 支持 dry-run 模式
必需依赖: call-webapi

⚡simply-opt (无脑对齐)

标准一键优化，将所有并发参数对齐到 partition 数

  • worker.size = partition
  • parallelismConfig 全对齐
  • 内存固定 4GB
  • buffer 固定 120
必需依赖: call-webapi

🧠normal-opt (智能诊断)

基于监控指标智能决策的调参方案

  • 分析 JobOffset/FullGc/Write
  • 智能内存决策 (4/5/6GB)
  • GC 高 → 增加内存
  • 写入跌0 → 诊断内存不足
必需依赖: call-webapi

🔬expert-opt (专家诊断)

深度诊断，查询日志识别具体错误类型

  • 查询 antlogs 异常日志
  • 识别 NoSuchTable/OOM/ODPS 冲突
  • 针对性优化策略
  • 需人工确认执行
必需依赖: call-webapi 可选依赖: antlogs MCP

📈global-view (健康检查)

批量任务健康检查，输出诊断结论

  • 查询三大指标
  • 判断延时趋势
  • 识别 GC 问题
  • 输出诊断结论
必需依赖: call-webapi

👑 黄金法则

所有优化必须遵守的规则：

  1. kepler.worker.size = parallelismConfig 所有算子 = Partition 数量（三者必须一致）
  2. job.worker.thread.number = 1
  3. 内存范围：2GB ~ 6GB（2147483648 ~ 6442450944 bytes）
  4. Buffer 范围：60 ~ 200
  5. 更新时 jarId 必须为 "588"（无论原始配置是什么）

📖 使用说明

快速开始

# 诊断并自动优化 ./dg.sh 197182 # 仅诊断，不执行优化 ./dg.sh 197182 -n # 强制模式，跳过确认 ./dg.sh 197182 -f # 分析最近2小时数据 ./dg.sh 197182 -d 120

单独使用各组件

# 批量健康检查 ./kepler_global_view_final.sh "197182,197183,197184" # 标准一键优化 ./kepler_simply_opt_simple.sh 197182 # 智能诊断优化 ./kepler_normal_opt.sh -t 197182 # 专家诊断（需人工确认） ./kepler_expert_opt.sh -t 197182
